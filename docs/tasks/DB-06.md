# DB-06: Add missing query performance indexes

**Priority:** P2  
**Area:** Schema  
**Status:** Open  
**Depends on:** —

## Why this matters

Three index gaps will affect real query patterns at Phase 1 launch:

1. **Venue and festival page queries** — `WHERE venue_id = X AND visibility = 'published' ORDER BY start_at` has no compound index; the planner filters on `idx_events_venue` then sorts. `WHERE festival_id = X AND visibility = 'published' ORDER BY start_at` has the same problem. Both are the primary queries for dedicated venue and festival pages.

2. **Server-side tag filtering** — `tags_display` is a comma-separated text field with no index. Any query filtering by tag content (`WHERE tags_display ILIKE '%techno%'`) sequential-scans the events table. This applies to a public search or filter API endpoint and to any coded-frontend query that uses the denormalised field rather than the junction table.

3. **Moderation queue** — the query "all draft events ready to publish" (`WHERE visibility = 'draft' AND confidence >= 60 AND needs_review = false`) has no supporting index. At 5,000 events/year this is fine, but it is a daily-use operator query and should be fast.

**Create migration `supabase/migrations/20260601000024_missing_indexes.sql`:**

```sql
-- Venue page: published events at a venue, ordered by date.
-- Pattern: WHERE venue_id = X AND visibility = 'published' ORDER BY start_at
create index idx_events_venue_date
  on events (venue_id, start_at)
  where visibility = 'published';

-- Festival page: published events in a festival, ordered by date.
-- Pattern: WHERE festival_id = X AND visibility = 'published' ORDER BY start_at
create index idx_events_festival_date
  on events (festival_id, start_at)
  where visibility = 'published' and festival_id is not null;

-- tags_display trigram index for server-side tag filtering.
-- Supports ILIKE and similarity queries on the comma-separated tags string.
-- Also useful for the coded-frontend path if tags_display is queried directly.
create index idx_events_tags_display_trgm
  on events using gin (tags_display gin_trgm_ops)
  where visibility = 'published';

-- Moderation queue: events ready to publish (confidence >= 60, not flagged, still draft).
-- Pattern: WHERE visibility = 'draft' AND confidence >= 60 AND needs_review = false
create index idx_events_ready_to_publish
  on events (confidence)
  where visibility = 'draft' and needs_review = false;
```
