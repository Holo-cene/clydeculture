# DB-10: Make seed data idempotent

**Priority:** P3  
**Area:** Schema, Ops  
**Status:** Open  
**Depends on:** —

## Why this matters

The `event_types` seed INSERT in the initial migration has no `ON CONFLICT DO NOTHING`
clause. `supabase db reset` drops and rebuilds, so this works for that case — but
running the SQL manually against an existing database (e.g., when a developer copies
the block from the docs, or applies a partial fix in the Supabase SQL editor) fails with
a unique constraint violation on `slug`. The recommended tag seed data in
`docs/reference/Schema v5 Assessment.md` has the same issue. Any seed script placed
in `supabase/seed.sql` that uses plain INSERTs will also fail on re-run. Making all
seed operations idempotent costs nothing and prevents avoidable friction.

---

## Prompt

You are building Clyde Culture. Read `docs/reference/SCHEMA_v5.sql` (the event_types
INSERT block) and `docs/reference/Schema v5 Assessment.md` (the tag seed data section)
before proceeding.

**Your task** is to make all seed INSERT statements in the schema and assessment
idempotent by adding `ON CONFLICT DO NOTHING` to each.

---

### Step 1 — Update the event_types seed in SCHEMA_v5.sql

Find the `insert into event_types` block and add `on conflict (slug) do nothing`:

```sql
insert into event_types (slug, label, sort_order) values
  ('live_music',        'Live Music',           1),
  ('club_night',        'Club Night / DJ',      2),
  ('comedy',            'Comedy',               3),
  ('theatre',           'Theatre',              4),
  ('arts_exhibition',   'Arts / Exhibition',    5),
  ('workshop',          'Workshop / Class',     6),
  ('talk_lecture',      'Talk / Lecture',        7),
  ('film',              'Film',                 8),
  ('family',            'Family',               9),
  ('sport',             'Sport',               10),
  ('community_meetup',  'Community / Meetup',  11),
  ('food_drink',        'Food & Drink',        12),
  ('other',             'Other',               99)
on conflict (slug) do nothing;
```

### Step 2 — Update the tag seed data in Schema v5 Assessment.md

Find both INSERT blocks in the "Tag Seed Data (Recommended Phase 1)" section of
`docs/reference/Schema v5 Assessment.md`.

For the category-specific tags INSERT, add `on conflict (slug) do nothing` before the
closing semicolon.

For the cross-category tags INSERT, add `on conflict (slug) do nothing` before the
closing semicolon.

### Step 3 — Create supabase/seed.sql

Create `supabase/seed.sql` containing the idempotent seed data for both `event_types`
and `tags`, copied from the updated versions above. This is the canonical location
Supabase CLI uses for seeding (`supabase db reset` runs this file after migrations).

The file should begin with a comment block:

```sql
-- Clyde Culture seed data
-- Idempotent: safe to re-run. All INSERTs use ON CONFLICT DO NOTHING.
-- Run via: supabase db reset (local) or supabase db seed (production)
```

Include both the `event_types` seed and the full recommended `tags` seed from the
assessment document.

### Step 4 — Update docs/OPERATIONS.md

In the Environments section, add a sentence to the Local environment description:

> Seed data (event types and tags) is applied automatically from `supabase/seed.sql`
> when running `supabase db reset`. The seed file is idempotent and safe to re-apply
> at any time.
