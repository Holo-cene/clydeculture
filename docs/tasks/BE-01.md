# BE-01: Resolve and document the execution runtime

**Priority:** P1  
**Area:** Runtime  
**Status:** Open  
**Depends on:** —

## Why this matters

The connectors are TypeScript/Node modules in a pnpm monorepo, but `INGESTION.md`
says the orchestrator runs as a Supabase Edge Function — which runs on Deno, not Node.
Node built-in imports (`from "crypto"`), npm packages installed via pnpm, and the
pnpm workspace package structure are all incompatible with Edge Functions without an
explicit bundling step that is not documented anywhere. Every connector, test, and CI
decision depends on knowing which runtime actually executes the code.

---

## Prompt

You are building Clyde Culture, a Glasgow cultural events aggregator. The ingestion
pipeline is described in `docs/INGESTION.md` and `docs/OPERATIONS.md`. The monorepo
layout is in `CLAUDE.md`. Read both files before proceeding.

The architecture has an unresolved runtime conflict: connectors are written as
Node/TypeScript pnpm workspace packages but `INGESTION.md` says execution happens
inside a Supabase Edge Function (Deno runtime). These are incompatible without an
explicit bundle step.

**Your task** is to resolve this conflict by writing an ADR that makes a concrete
decision, then updating the relevant docs to reflect that decision.

1. Create `docs/decisions/0002-ingestion-runtime.md` using the template at
   `docs/decisions/0000-adr-template.md`. The ADR must answer:
   - Which option from `docs/OPERATIONS.md` is the chosen execution host
     (Option A: Edge Functions, Option B: external Node worker, or Option C: pg_cron)
   - If Option A: how pnpm workspace packages are bundled into the Edge Function
     at deploy time, and whether `node:` prefix shims are required for Node built-ins
   - If Option B: which external host (GitHub Actions, Render, Railway) and how the
     job is triggered and monitored
   - How the 150-second Edge Function wall-clock limit is handled if Option A is chosen
   - Whether HTML scrapers using `npm:cheerio` are compatible with the chosen runtime
   
2. Update the "Scheduled Ingestion" section of `docs/OPERATIONS.md` to remove
   hedging language and state the chosen option as fact, with a reference to the ADR.

3. Update `docs/INGESTION.md` to replace "a Supabase Edge Function invoked by a cron
   trigger" with the actual chosen mechanism, and add a sentence about how workspace
   packages are made available to the execution host.

**Do not write any connector code.** Documentation and the ADR file only.
