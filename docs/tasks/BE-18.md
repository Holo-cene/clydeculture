# BE-18: Set up connector test infrastructure

**Priority:** P3  
**Area:** Testing  
**Status:** Open  
**Depends on:** BE-01

## Why this matters

`docs/CONNECTOR_GUIDE.md` describes testing via a scratch file run with `npx tsx`.
This is adequate for smoke-testing a single connector during development but provides
no automated regression coverage, no CI enforcement, and no shared test utilities for
things every connector needs (mock HTTP responses, valid `RawEvent` shape assertion,
`externalId` stability check). Without a test framework in place before connectors are
built, tests will either never be written or be written inconsistently across connectors.

---

## Prompt

You are building Clyde Culture. Read `docs/CONNECTOR_GUIDE.md` and `CLAUDE.md` before
proceeding. The monorepo uses pnpm workspaces. Connectors live in
`packages/connectors/`. Read the monorepo layout in `CLAUDE.md`.

**Your task** is to set up a lightweight test infrastructure for the connectors package
and update the connector guide to make testing a first-class part of the build process.

**Step 1 — Add vitest to `packages/connectors`:**

If `packages/connectors/package.json` exists:

```json
{
  "devDependencies": {
    "vitest": "^2.0.0",
    "msw": "^2.0.0"
  },
  "scripts": {
    "test": "vitest run",
    "test:watch": "vitest"
  }
}
```

`msw` (Mock Service Worker) provides HTTP request mocking without monkey-patching
`fetch` — reliable for testing connectors that make HTTP calls.

Add `packages/connectors/vitest.config.ts`:

```ts
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    environment: 'node',
    include: ['src/**/*.test.ts'],
    coverage: { reporter: ['text'] },
  },
});
```

**Step 2 — Create shared test utilities at `packages/connectors/src/testing/`:**

`packages/connectors/src/testing/assert-raw-event.ts`:
```ts
import { expect } from 'vitest';
import type { RawEvent } from '../connector';

// Asserts that an item is a well-formed RawEvent.
// Use in all connector tests to enforce the interface contract.
export function assertValidRawEvent(item: RawEvent): void {
  expect(item.externalId).toBeTruthy();
  expect(item.externalUrl).toBeTruthy();
  expect(item.title).toBeTruthy();
  expect(item.raw).toBeDefined();
  // externalUrl must look like a URL
  expect(() => new URL(item.externalUrl)).not.toThrow();
}

// Asserts that running the connector twice produces the same externalIds.
export function assertStableIds(run1: RawEvent[], run2: RawEvent[]): void {
  const ids1 = new Set(run1.map(r => r.externalId));
  const ids2 = new Set(run2.map(r => r.externalId));
  for (const id of ids1) {
    expect(ids2.has(id), `externalId "${id}" missing on second run`).toBe(true);
  }
}
```

**Step 3 — Create an example test file for the RSS connector shape:**

`packages/connectors/src/rss/glasgow-art-map/index.test.ts` (or a generic shape test):

```ts
import { describe, it, expect } from 'vitest';
import { assertValidRawEvent } from '../../testing/assert-raw-event';
// Import the connector once it exists:
// import { glasgowArtMapConnector } from './index';

describe('RawEvent shape contract', () => {
  it('every item satisfies the RawEvent interface', () => {
    // Synthetic fixture — replace with connector.run() once the connector is built
    const fixture = {
      externalId: 'abc123',
      externalUrl: 'https://example.com/event/1',
      title: 'Test Event',
      raw: { title: 'Test Event' },
    };
    assertValidRawEvent(fixture);
  });
});
```

**Step 4 — Update `docs/CONNECTOR_GUIDE.md`** section 8 ("Test locally"):

Replace the scratch-file instructions with:

> Run tests with `pnpm --filter @clydeculture/connectors test`. Use `msw` to mock
> HTTP responses for deterministic unit tests. Use `assertValidRawEvent()` from
> `packages/connectors/src/testing/assert-raw-event.ts` in every connector test.
> The CI pipeline runs `pnpm test` on every push — a PR with a new connector must
> include at least one test that asserts `parsedCount > 0` against a mocked response
> and that `externalId` values are stable across two calls.
