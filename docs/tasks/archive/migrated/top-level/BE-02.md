# BE-02: Implement deletion detection in the orchestrator

**Priority:** P1  
**Area:** Ingestion  
**Status:** Open  
**Depends on:** BE-01

## Why this matters

The six-step orchestrator in `docs/INGESTION.md` upserts records that appear in a run
but has no step to mark records that *disappeared*. Without a deletion step,
`external_events.is_deleted` is never set to `true`, events removed from upstream APIs
stay published indefinitely, and the lifecycle described in the data model ("when an
external record disappears from the source, `is_deleted = true` propagates to the
canonical event") never actually happens.

---

## Prompt

You are building Clyde Culture. Read `docs/INGESTION.md`, `docs/DATA_MODEL.md`, and
`docs/reference/SCHEMA_v5.sql` before proceeding.

The orchestrator in `docs/INGESTION.md` has six steps. There is no step that marks
records absent from the current run as deleted, and no Postgres function that propagates
deletion to the canonical `events` table. Add both.

**Step 1 — Update the orchestrator spec in `docs/INGESTION.md`:**

Add a step 7 to the numbered list in the "Scheduled-job model" section:

> 7. After each successful run (status = 'success' or 'partial'), marks stale records as
>    deleted: sets `is_deleted = true` on all `external_events` rows for this source
>    where `last_seen_at < run_started_at AND is_deleted = false`. Records absent from the
>    current run have not had `last_seen_at` refreshed and are therefore stale.

Also add a subsection titled "Deletion propagation" that specifies the multi-source rule:
a canonical event moves to `visibility = 'archived'` only when every non-deleted
contributing `external_events` row (`is_deleted = false`) has been marked deleted —
i.e., no live source still carries the event.

**Step 2 — Add a Postgres function to `supabase/migrations/`:**

Create a new migration file `supabase/migrations/20260601000001_deletion_detection.sql`
containing two functions:

```sql
-- Marks stale external records for one source after a completed run.
-- Call with the source_id and the run's started_at timestamp.
create or replace function mark_stale_deleted(
  p_source_id uuid,
  p_run_started_at timestamptz
) returns integer as $$
-- Sets is_deleted = true on rows last_seen_at < p_run_started_at
-- Returns count of rows marked deleted
$$ language plpgsql;

-- Propagates is_deleted to the canonical event when all sources agree.
-- Call after mark_stale_deleted.
create or replace function propagate_deletion(p_event_id uuid)
returns void as $$
-- If every external_events row for this event has is_deleted = true,
-- set events.visibility = 'archived'
$$ language plpgsql;
```

Implement both functions fully. `propagate_deletion` should only archive the event when
`NOT EXISTS (SELECT 1 FROM external_events WHERE event_id = p_event_id AND is_deleted = false)`.

**Step 3 — Update `docs/DATA_MODEL.md`:**

Add a sentence to the `external_events` section clarifying that `is_deleted` is set by
`mark_stale_deleted()`, not by a trigger, and linking to the `INGESTION.md` deletion
propagation subsection.
