# BE-19: Externalise the confidence-score publishing threshold

**Priority:** P3  
**Area:** Normalisation  
**Status:** Open  
**Depends on:** BE-03

## Why this matters

`docs/DATA_MODEL.md` and `docs/PUBLISHING.md` refer to "the confidence threshold
(currently 60)" as the gate between `draft` and `published`. This value is
hardcoded in prose — it appears in no config table, no environment variable, and no
schema column. If the threshold needs adjusting (e.g., to 70 after audit, or lowered
to 50 for a specific source tier), a developer must hunt through the codebase and
update every reference. It is also unclear whether the threshold is global or
per-source, which means different connectors cannot have different quality bars.

---

## Prompt

You are building Clyde Culture. Read `docs/PUBLISHING.md`, `docs/DATA_MODEL.md`, and
`docs/reference/SCHEMA_v5.sql` before proceeding.

**Your task** is to externalise the confidence threshold so it is configurable without
a code change, and support per-source overrides.

**Step 1 — Add `confidence_threshold` to the `sources` table:**

Create migration `supabase/migrations/20260601000013_confidence_threshold.sql`:

```sql
-- Per-source confidence threshold. Events from this source must meet or exceed
-- this score to be eligible for visibility = 'published'.
-- Global default: 60. Set lower for high-trust manual sources (100 = always trusted),
-- or higher for Tier 4 enrichment sources where confidence is naturally lower.
alter table sources
  add column if not exists confidence_threshold smallint not null default 60
  check (confidence_threshold between 0 and 100);
```

This makes the threshold both per-source configurable and has a sensible default (60)
that matches the current documented value.

**Step 2 — Add a `platform_config` table for the global default:**

In the same migration:

```sql
-- Single-row configuration table for global platform settings.
-- Currently only stores the default confidence threshold; extend as needed.
create table platform_config (
  id                        integer primary key default 1
    check (id = 1),           -- enforces single-row
  default_confidence_threshold smallint not null default 60
    check (default_confidence_threshold between 0 and 100),
  updated_at                timestamptz not null default now()
);

insert into platform_config (default_confidence_threshold) values (60);

alter table platform_config enable row level security;
-- No public policy; read via service role.
```

**Step 3 — Update `docs/PUBLISHING.md`:**

In "The publishing boundary" section, replace "the configured minimum" with:

> The per-source confidence threshold (stored in `sources.confidence_threshold`,
> defaulting to 60). To change the threshold for a specific source:
> `UPDATE sources SET confidence_threshold = 70 WHERE slug = 'ticketmaster';`
> To change the global default for new sources:
> `UPDATE platform_config SET default_confidence_threshold = 70;`

**Step 4 — Update `docs/DATA_MODEL.md`:**

- Add `confidence_threshold` to the `sources` table description.
- Add `platform_config` to the Table Map.
- Update the `confidence` field description on `events` to reference
  `sources.confidence_threshold` rather than the magic number 60.
