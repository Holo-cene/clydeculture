# BE-12: Fix unbounded loop in auto_create_venue

**Priority:** P2  
**Area:** Schema  
**Status:** Open  
**Depends on:** —

## Why this matters

`auto_create_venue()` generates a slug and resolves collisions with:

```sql
while exists (select 1 from venues where slug = v_slug) loop
  v_slug := v_slug || '-' || floor(random() * 1000)::text;
end loop;
```

Each iteration appends a random 0–999 suffix. On repeated collision (possible if many
venues share a common name fragment), the loop compounds suffixes (`venue-342-17-891`)
and has no termination guarantee. A sequential counter (`-2`, `-3`, `-4`) matches the
slug collision convention used for `events` and makes the loop predictable and bounded.

Replace the function with a version that uses a deterministic sequential counter:

```sql
create or replace function auto_create_venue(
  p_venue_name text,
  p_source_url text default null
)
returns uuid as $$
declare
  v_venue_id   uuid;
  v_base_slug  text;
  v_slug       text;
  v_counter    integer := 2;
begin
  -- Build base slug from name: lowercase, alphanumeric + hyphens
  v_base_slug := lower(trim(regexp_replace(
    regexp_replace(p_venue_name, '[^[:alnum:][:space:]]', '', 'g'),
    '[[:space:]]+', '-', 'g'
  )));

  v_slug := v_base_slug;

  -- Resolve slug collisions with a sequential counter (matches events slug convention)
  while exists (select 1 from venues where slug = v_slug) loop
    v_slug := v_base_slug || '-' || v_counter::text;
    v_counter := v_counter + 1;
    -- Safety: if somehow 100 collisions occur, raise an error rather than looping forever
    if v_counter > 100 then
      raise exception 'auto_create_venue: could not generate unique slug for "%"', p_venue_name;
    end if;
  end loop;

  insert into venues (name, slug, status, auto_created, needs_review, website)
  values (p_venue_name, v_slug, 'pending', true, true, p_source_url)
  returning id into v_venue_id;

  insert into venue_aliases (venue_id, alias, normalised_alias)
  values (v_venue_id, p_venue_name, lower(trim(p_venue_name)))
  on conflict (normalised_alias) do nothing;

  return v_venue_id;
end;
$$ language plpgsql;
```

**No documentation update is needed** — the function behaviour is unchanged from the
caller's perspective; only the internal slug-generation strategy changes.
