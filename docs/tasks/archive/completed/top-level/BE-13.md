**Priority:** P2  
**Area:** Normalisation

Remaining scope: create `ConfidenceInputs` TypeScript type (tracked in C2).  
**Depends on:** BE-03

## Why this matters

`events.confidence_inputs jsonb` is referenced in `docs/DATA_MODEL.md` and
`docs/PUBLISHING.md` as storing "a structured breakdown of score components"
(source tier, field completeness, venue resolution success, cross-source
corroboration, category confidence). The JSON schema for this field is nowhere
defined. Any agent or engineer implementing the normalisation pipeline will invent
their own structure, making `confidence_inputs` unqueryable and inconsistent across
connectors.

**Step 1 — Add to `docs/NORMALISATION.md`** a section titled
"Confidence score — inputs JSON structure":

Define the canonical structure:

```json
{
  "source_tier": 1,
  "has_title": true,
  "has_start_at": true,
  "has_venue_name": true,
  "venue_resolved": true,
  "venue_auto_created": false,
  "has_end_at": false,
  "has_image": true,
  "has_summary": true,
  "cross_source_corroborated": false,
  "category_confidence": "mapped",
  "score_components": {
    "base": 60,
    "field_bonus": 30,
    "corroboration_bonus": 0,
    "category_penalty": 0,
    "final": 90
  }
}
```

Field definitions:
- `source_tier`: integer 1–4 from `sources.tier`
- `has_title` / `has_start_at` / `has_venue_name` / `has_end_at` / `has_image` / `has_summary`: boolean; true if the field was non-null after extraction
- `venue_resolved`: true if `resolve_venue()` found an existing match; false if `auto_create_venue()` was called
- `venue_auto_created`: true if a new stub venue was created (distinct from resolved)
- `cross_source_corroborated`: true if another `external_events` row from a different source already points to this canonical event
- `category_confidence`: `"mapped"` (found in `source_type_category_map`), `"keyword"` (matched a keyword rule), or `"fallback"` (assigned `other`)
- `score_components`: the computed sub-scores that sum to `final`, matching the formula in `docs/NORMALISATION.md`

**Step 2 — Create `packages/shared/src/types/confidence.ts`:**

```ts
export type CategoryConfidence = 'mapped' | 'keyword' | 'fallback';

  has_venue_name: boolean;
  venue_resolved: boolean;
  venue_auto_created: boolean;
  has_end_at: boolean;
  has_image: boolean;
  has_summary: boolean;
  cross_source_corroborated: boolean;
  category_confidence: CategoryConfidence;
  score_components: {
    base: number;
    field_bonus: number;
    corroboration_bonus: number;
    category_penalty: number;
    final: number;
  };
}
```
