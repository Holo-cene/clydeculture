# BE-14: Document and implement primary_source_id update logic

**Priority:** P2  
**Area:** Normalisation  
**Status:** Open  
**Depends on:** BE-03

## Why this matters

`events.primary_source_id` references the source that is considered the authority for
this canonical event's field values. The field exists in the schema but its assignment
and update logic is nowhere documented. When does it get set? When Ticketmaster and
Skiddle both carry the same event and their records merge via `dedupe_key`, which
becomes primary? When the primary source stops returning an event, does `primary_source_id`
update to the next-highest-tier source? Without specified rules, the field is effectively
untrustworthy and any logic built on it will be inconsistent.

**Your task** is to specify the `primary_source_id` assignment rules and implement them

**Step 1 — Update `docs/DEDUPLICATION.md`:**

Add a subsection "Primary source selection" after the "Canonical record preference: API
over scraped" section:

> `primary_source_id` on `events` identifies which `sources` row is the current
> authority for the event's canonical field values. Selection rules, in order:
>
> 1. Prefer the lowest `sources.tier` value (Tier 1 > Tier 2 > Tier 3 > Tier 4).
> 2. Within the same tier, prefer the most recently fetched record
>    (`external_events.last_seen_at DESC`).
> 3. If all contributing external records are marked `is_deleted = true`,
>    `primary_source_id` is set to null and the event is archived (see BE-02).
>
> `primary_source_id` is recomputed whenever an `external_events` row that links to
> this canonical event is added, updated, or marked deleted. It is stored on `events`
> to avoid a JOIN on every publish query.

**Step 2 — Create migration `supabase/migrations/20260601000010_primary_source.sql`:**

Add a function `elect_primary_source(p_event_id uuid) returns uuid`:

```sql
create or replace function elect_primary_source(p_event_id uuid)
returns uuid as $$
-- Returns the source_id of the highest-priority live source contributing to this event.
-- Priority: lowest tier first; tie-break by most recent last_seen_at.
-- Returns null if no live external records exist.
declare
  v_source_id uuid;
begin
  select ee.source_id
  into v_source_id
  from external_events ee
  join sources s on s.id = ee.source_id
  where ee.event_id = p_event_id
    and ee.is_deleted = false
  order by s.tier asc, ee.last_seen_at desc
  limit 1;

  return v_source_id;
end;
$$ language plpgsql stable;
```

Also add an `AFTER UPDATE ON external_events` trigger that recomputes `primary_source_id`
on the linked canonical event whenever `is_deleted` or `last_seen_at` changes:

```sql
create or replace function trigger_reelect_primary_source()
returns trigger as $$
begin
  if new.event_id is not null and (
    new.is_deleted   is distinct from old.is_deleted or
    new.last_seen_at is distinct from old.last_seen_at
  ) then
    update events
    set primary_source_id = elect_primary_source(new.event_id)
    where id = new.event_id;
  end if;
  return new;
end;
$$ language plpgsql;

create trigger reelect_primary_source_after_update
  after update on external_events
  for each row execute function trigger_reelect_primary_source();
```
