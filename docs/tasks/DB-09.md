# DB-09: Fix SPEC.md stale field definitions

**Priority:** P3  
**Area:** Docs  
**Status:** Open  
**Depends on:** —

## Why this matters

`SPEC.md §8` contains four field definitions that diverge from `SCHEMA_v5.sql` and
`DATA_MODEL.md`. The spec correctly notes "Where this section and the v5 schema differ,
the v5 schema wins" — but the caveat is easy to miss and the drift has already caught
reviewers. An AI agent or new contributor reading SPEC.md as their first document will
write code referencing `event_type` (not `event_type_id`), `tags text[]` (not the
junction table), `visibility = 'cancelled'` (removed in v5), and `status = 'partial'`
(renamed to `degraded`). All four produce either runtime errors or silently bad data.

This is documentation only — no schema changes.

---

## Prompt

You are building Clyde Culture. Read `docs/reference/SPEC.md` §8 (Data Model) and §9
(Deduplication Strategy), `docs/reference/SCHEMA_v5.sql`, and `docs/DATA_MODEL.md`
before proceeding. Do not modify any migration files or source code.

**Your task** is to correct four stale field definitions in `docs/reference/SPEC.md`.

---

### Fix 1 — events.visibility enum values

In `SPEC.md §8`, the events table row for `visibility` currently reads:

> `visibility | enum | draft / published / hidden / cancelled`

Replace with:

> `visibility | text | `draft`, `published`, `hidden`, `archived` — see DATA_MODEL.md §"visibility vs. confidence / needs_review". Note: `cancelled` was removed in v5; use `events.availability = 'cancelled'` instead.`

---

### Fix 2 — events.tags column

In `SPEC.md §8`, the events table has a row:

> `tags | text[] | Granular tags: techno, improv, sculpture, etc.`

Replace with:

> `tags | (junction table) | Tags are stored in the `event_tags` junction table (`event_id`, `tag_id`). The denormalised `tags_display text` field on `events` holds a comma-separated string for Webflow. See DATA_MODEL.md §event_tags.`

---

### Fix 3 — events.event_type column

In `SPEC.md §8`, the events table has a row:

> `event_type | enum | See taxonomy in Section 4`

Replace with:

> `event_type_id | smallint FK → event_types | Normalised FK. The denormalised `event_type_label text` field holds the display string for Webflow. See DATA_MODEL.md §Classification.`

---

### Fix 4 — sources.status values

In `SPEC.md §8` the sources table row for `status` currently reads:

> `status | ok / partial / broken / disabled`

Replace with:

> `status | `ok`, `degraded`, `broken`, `disabled` — note: `partial` was renamed to `degraded` in v5. `degraded` is set automatically by break detection when `parsed_count` drops >70% below the 14-day median.`

---

### Add a warning banner

At the top of the `## 8. Data Model` section in `SPEC.md`, add:

> ⚠️ **This section is illustrative.** The authoritative schema is
> `docs/reference/SCHEMA_v5.sql` (20 tables). The authoritative prose description is
> `docs/DATA_MODEL.md`. Where this section and either of those documents differ, those
> documents win. This section is kept for high-level orientation only and may lag
> schema changes.
